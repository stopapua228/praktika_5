Этап 1. Минимальный прототип с конфигурацией
Создал минимальное CLI-приложение и сделал его настраиваемым.
Реализовано:
    1. Настраиваемые параметры:
* – Имя анализируемого пакета.
* – URL-адрес репозитория или путь к файлу тестового репозитория.
* – Режим работы с тестовым репозиторием.
* – Имя сгенерированного файла с изображением графа (формат PNG).
  * При запуске программа выводит все параметры в формате ключ=значение.
* Реализована обработка ошибок для всех параметров:
– Проверяется наличие и непустота имени пакета.
– В тестовом режиме проверяется существование указанного файла.
– При обычном режиме проверяется валидность URL (http/https).
– Контролируется, что файл вывода оканчивается на .png и есть права на запись в каталог.
* Разработан демонстрационный скрипт для проверки корректных и ошибочных запусков, который сохраняет результаты в папку examples.
